========================================================
What is an EBS Volume?
========================================================

Amazon Elastic Block Store (EBS) is a block storage service
provided by AWS.

It works like a virtual hard drive that can be attached to
EC2 instances.

EBS volumes are commonly used for:

- Operating system storage
- Application data storage
- Database storage
- Backup and recovery purposes

========================================================


Step 3: Create the EBS Volume

Command Used:

aws ec2 create-volume \
--volume-type gp3 \
--size 2 \
--availability-zone us-east-1a


Explanation:

--volume-type gp3
    Creates a General Purpose SSD (gp3) volume.

--size 2
    Creates a volume with 2 GiB storage capacity.

--availability-zone us-east-1a
    Places the volume in the selected AWS Availability Zone.


Result:

The EBS volume was created successfully.

Size:

========================================================


Step 4: Add Name Tag to the Volume

Problem:

The volume was created successfully, but AWS resources do not
automatically have names.

The task required the volume name to be:

nautilus-volume


Solution:

AWS uses tags to assign names to resources.


Command Used:

aws ec2 create-tags \
--resources vol-0a2688b250fc2d7c4 \
--tags Key=Name,Value=nautilus-volume



Result:

The required name tag was added successfully.



========================================================


Step 5: Verify the Volume Configuration

--filters Name=tag:Name,Values=nautilus-volume

- Availability Zone
